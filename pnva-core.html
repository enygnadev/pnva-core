Architecture

Execute by cause, not by temporal habit.

PNVA-Core models computation as a field of observable state changes that generate events, tension, collapse decisions, restricted execution and proof logs.

Core Chain

state -> event -> tension -> collapse -> execution -> proof

State

Current observable condition of a system entity or field.

Event

Detectable state change that enters the PNVA field.

Tension

Operational pressure generated by event strength, state gradient, cost and memory.

Collapse

Decision point where tension crosses threshold and action is authorized.

Execution

Restricted action performed only after causal justification.

Proof

Auditable JSON record explaining why the action was taken or ignored.
